# 0008: Exports — PDF + CSV + JSON

**Blocked by:** 0005
**Blocks:** 0009
**Requirements addressed:** R-015 (export reports)
**Demoable:** From a single-review detail screen the reviewer clicks Export → PDF or JSON and gets the file; from a batch summary the reviewer can download a CSV summary, a per-field-results CSV, all-PDFs ZIP, all-JSON ZIP.
**Estimated effort:** 4-5h

## Acceptance criteria
- [ ] R-015 (PDF — per-label review report)
  - `app/api/render-pdf/route.ts` accepts a Review JSON, returns a PDF
  - Server-side via `@react-pdf/renderer`
  - Layout: header (proofLens + version) → review metadata (reviewer, date, beverage) → image thumbnail → expected vs extracted field-by-field table → overall verdict → human override section (if any) → reviewer notes → signature line ("This review was generated by proofLens. Final decision is the reviewer's responsibility.")
- [ ] R-015 (CSV — batch summary)
  - One row per label: `id, filename, brand, beverage, overall_status, completed_at, reviewer, has_overrides, processing_time_ms`
  - Generated via `papaparse` server-side from the batch + reviews JSON
- [ ] R-015 (CSV — per-field detail)
  - One row per (label × field): `review_id, filename, field_name, expected, extracted, status, confidence, ai_status, human_status, override_reason`
- [ ] R-015 (JSON)
  - Single review: full Review object
  - Batch: `{ batch: Batch, reviews: Review[] }`
- [ ] R-015 (Batch ZIP)
  - All-PDFs: stream PDFs into ZIP via `archiver`
  - All-JSONs: same shape, JSON files
  - Streamed response; no temp files
- [ ] Export menus
  - Single review: `Export → [PDF] [JSON]`
  - Batch summary: `Export → [Summary CSV] [Per-field CSV] [All PDFs (zip)] [All JSON (zip)]`
- [ ] All quality gates green
- [ ] `STUDY_GUIDE.md` updated: "How we generate PDFs server-side" + "Why ZIP exports stream"

## Notes
- PDF embeds the 256px thumbnail (not the original); aligns with IT note (no original retention) — note in the PDF footer.
- The `@react-pdf/renderer` setup may need a custom font file (Inter) — embed as a static file.
- ZIP export streams; do not buffer in memory (250 PDFs × ~80KB each = ~20MB; manageable but stream is cleaner).
- Each export action emits a small toast confirmation.
- PDFs include the canonical § 16.21 string in a footer note for transparency: "Government warning text was checked against 27 CFR § 16.21 (verified 2026-04-30)".
