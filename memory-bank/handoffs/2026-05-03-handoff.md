## Recommended next step

**Phase 6 of the full-review plan: Vercel-sim test infrastructure.**

Why this next (vs Phase 2/3/4): Phase 6 is the regression net.
Building it before the rest of the audit means every bug surfaced
in Phases 2-5 gets a regression test added in the same commit.
Without Phase 6, there's a real risk of ANOTHER Vercel-only bug
slipping through (the same pattern that produced the function-phrase
false-warn and the dead bbox).

**What Phase 6 builds:**

1. A new vitest suite at `lib/verify/pipeline.production-env.test.ts`
   — every grader tested against `(extracted, rawText='just the
   gov warning', words=[])` and asserts the verdict makes sense.
2. A new playwright project named `production-sim` in
   `playwright.config.ts`. Sets `webServer.env.VERCEL='1'` so the
   dev server triggers the `skipTesseract` branch in
   `app/api/extract-label/route.ts:234`.
3. A subset of e2e specs run under that project — at least: queue,
   single-label, override-and-history, scenario-switch, verification.
4. Quality-gate addition: both new suites must pass before push.

**Files to create:**
- `lib/verify/pipeline.production-env.test.ts`
- (no new e2e specs needed — existing ones get re-run under the
  new playwright project)

**Files to modify:**
- `playwright.config.ts` — add the `production-sim` project
  (mirror the existing `chromium` block but add
  `webServer.env.VERCEL: '1'`).
